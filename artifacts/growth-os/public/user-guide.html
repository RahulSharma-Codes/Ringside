Section 1

Introduction

Ringside is Manipal Group's proprietary M&A deal intelligence platform. It gives the Corporate Development team a single, structured workspace to track every acquisition target — from first identification through to deal closing — replacing scattered spreadsheets, email threads, and disparate tools with one authoritative system of record.

The platform is designed around the reality of how deals actually move: iteratively, with many stakeholders, long time horizons, and a constant need to prioritise attention across a live pipeline. Ringside makes the entire pipeline visible at a glance, ensures no deal falls through the cracks, and provides the analytical depth required for IC-level decision-making.

🗺️ How Ringside is organised

📊
Pipeline Management
Track every deal through a structured 12-stage pipeline. Switch between a list view and a visual Kanban board. Filter, search, and export.
🎯
Target Intelligence
A dedicated workspace per deal covering interactions, actions, diligence checklists, documents, valuation, IC sessions, stakeholders, and compliance.
📋
Review Cadence
Weekly Review, Diligence Review, and Document Review pages provide structured summaries for recurring governance meetings.
🤖
AI Intelligence
The AI Copilot answers questions about your pipeline, parses meeting notes, and generates ready-to-send IC briefs — all grounded in live deal data.
📈
Analytics
Pipeline funnel, time-in-stage analysis, win/loss breakdown, and origination channel performance — all from your actual deal history.
🔐
Access Control
Non-admin users see only the deals they've been granted access to. Admins manage users, roles, and deal-level visibility from the Admin Console.

🔑 Access & login

Ringside is invitation-only. You will receive an email invite from your administrator. Clicking the invite link takes you through a one-time account setup where you choose your password. Subsequent logins use your email address and password at the Ringside sign-in page.

ℹ️
If you need an account, contact your Ringside administrator. Self-registration is not available. All activity on the platform is logged.
Section 2

Quick Start

This section gets you productive in under 10 minutes. Read the full guide to understand each feature in depth.

  1. Sign in Go to the Ringside URL, enter your email and password, and click Sign In. You land on the Dashboard.
  2. Check the Dashboard for immediate priorities The KPI strip at the top shows your pipeline health. Scroll down to Top Priority Deals to see which deals need action today.
  3. Open the Pipeline to browse all deals Click Pipeline in the sidebar. Use the filters to narrow by stage, tier, or owner. Click any deal row to open its full detail page.
  4. Add a new target In Pipeline, click the New button (top right). Fill in the project code name, sector, priority tier, and key details. Click Create Target.
  5. Log an interaction on a deal Open any target. On the Log tab, click Log Interaction. Select the type (Call, Meeting, Email…), add your notes, and save. This timestamps the last contact automatically.
  6. Run your Weekly Review Click Weekly Review in the sidebar for a structured summary of must-win deals, overdue actions, recent stage changes, and deals with no recent activity.
  7. Ask the AI Copilot a question Click AI Copilot in the sidebar. Type a question like "Which Must-Win deals have no open actions?" and the AI responds with live pipeline data.
💡
Press ⌘K (Mac) or Ctrl+K (Windows) from anywhere in the app to open the Command Palette — a fast way to jump to any deal or page without using the sidebar.

Section 5

Pipeline

The Pipeline page is the central directory of all deals. Every target in the system lives here. You can view deals as a sortable table or as a visual Kanban board.

🔍 Filters & search

Filters across the top let you narrow the view by:

  • Search — fuzzy search by deal name or target code
  • Stage — any of the 12 active pipeline stages
  • Tier — Must-Win, Priority 1, Priority 2, Watchlist, On Hold, Dropped
  • Owner — filter to a specific deal owner
  • Country — geography filter
  • Type — deal type (Acquisition, Minority Investment, Divestiture, JV, Partnership, Strategic Alliance)
  • Attn — show only deals flagged as Needs Attention
  • My Deals / All Deals — toggle between your deals and the full pipeline

Click Clear Filters to reset all filters at once.

📋 List view

The list view shows all deals in a sortable table. Key columns:

🗂️ Kanban board view

Switch to the Board view using the toggle in the top-right corner. Each column represents a pipeline stage. Deal cards show the target name, tier badge, health indicator, and days in stage.

Drag-and-drop stage changes

In Kanban view, active deal cards can be dragged between columns. When you drop a card on a new stage column, a Confirm Stage Change dialog appears — select a reason (or type a custom one) and confirm. The deal moves to the new stage and the change is recorded in the audit trail.

ℹ️
Cards in the On Hold, Dropped, and Rejected columns are not draggable — click them to open the deal detail page.

➕ Creating a new target

  1. Click the New buttonTop-right of the Pipeline page.
  2. Fill in Core IdentityProject Code Name (internal alias), Identifier (target code), Legal Entity Name, and a Strategic Rationale.
  3. Set CategorisationSector, Subsector, Country, Region, Deal Type, and Business Unit.
  4. Assign TeamDeal Owner, Deal Champion, Executive Sponsor, Sourcing Channel, Sourcing Firm.
  5. Score the dealDrag the five scoring sliders: Strategic Fit, Synergy Potential, Financial Attractiveness, Process Maturity, Risk Penalty.
  6. Set Priority Tier and ConfidentialityChoose a tier and toggle strict confidentiality if required.
  7. Click Create TargetThe deal is created at the Sourcing stage and appears in the Pipeline.

📤 Export & Import

Use the Export button to download the current filtered view as a spreadsheet. Use Import to bulk-upload deals via the 5-step Import Wizard (see Section 14).

🗃️ Overview tab

The Overview tab contains the deal's foundational information: the Strategic Rationale, key facts (sector, deal type, ownership structure, sourcing details), the Score Card (each scoring dimension visualised as a bar with the overall composite score), and — for closed or dropped deals — a Verdict section capturing the outcome and rationale.

💬 Log tab (Interaction Log)

Every touchpoint with or about the target is recorded here: calls, meetings, emails, and general notes. The log is sorted chronologically and drives the Last Contact date shown in the Pipeline list and Dashboard.

Logging an interaction

  1. Click Log InteractionButton at the top of the Log tab, or tap the Log quick-action in the mobile bottom bar.
  2. Select the interaction typeCall, Meeting, Email, Note, or other type as configured.
  3. Write your notesUse the rich text editor for structured notes with formatting.
  4. SaveThe interaction is timestamped and added to the log. Last Contact date updates immediately.
💡
Use the AI Copilot's meeting notes parser if you have raw meeting notes you'd like automatically structured before logging. See Section 12.

✅ Actions tab

The Actions tab holds the task list for this specific deal — next steps, outstanding items, and follow-ups assigned to team members. Actions here appear in the pipeline-wide Actions Command Center.

Adding an action

  1. Click Add ActionOr tap the + quick-action in the mobile bottom bar.
  2. Fill in the fieldsDescription (rich text), Priority (Critical / High / Medium / Low), Due Date, Owner.
  3. SaveThe action appears in the list and in the Actions Command Center.

You can mark actions Complete or reopen them using the quick buttons on each action card. Completed actions are shown separately at the bottom of the list.

🔎 Diligence tab

The Diligence tab contains a structured due-diligence checklist organised into 10 workstreams. A Readiness Score card at the top shows overall completion percentage, blocked items, overdue items, and missing workstreams.

Workstreams available:

  • Commercial · Financial · Legal · Tax · HR · Technology · Operations · Integration · ESG · Regulatory

Adding a diligence item

  1. Click Add ItemOpens the Add Diligence Item dialog.
  2. Select the workstreamCommercial, Financial, Legal, etc.
  3. Fill in the detailsDescription, Owner, Due Date, Priority (Critical/High/Medium/Low), Status (Open/In Progress/Blocked/Completed), Notes (rich text), and optional Evidence Links.
  4. SaveThe item appears in the correct workstream section and updates the Readiness Score.

Each workstream section is collapsible. Quick Complete and Reopen buttons appear on each item card. Items can be edited or deleted at any time.

⚠️
Diligence items are separate from Actions. They appear in the Diligence Review page, not the Actions Command Center.

📁 Documents tab

The Documents tab is a deal-level document vault. Files are uploaded to secure cloud storage and organised by category. Each document record has a title, category (e.g. CIM, Financial Model, Legal, NDA, Management Presentation), classification level, owner, and optional notes.

Supported file types

PDF, Word (.docx), Excel (.xlsx), PowerPoint (.pptx), CSV, JPEG, PNG, TIFF, ZIP, MP4 — up to 25 MB per file.

Uploading a document

  1. Click Upload DocumentFill in the title, category, and classification. Then attach your file.
  2. Click UploadThe file is stored securely and a preview link is generated.
🔒
Highly Restricted documents can only be downloaded by Admins or the verified deal owner. All other users see the document record but cannot download the file.

💰 Valuation tab

The Valuation tab is used to record and track financial modelling outputs and price discussions for the deal — entry price assumptions, valuation multiples, IRR targets, and related financial metrics as the deal progresses.

✨ Synergies tab

The Synergies tab captures the strategic value creation thesis for the acquisition — revenue synergies, cost synergies, capability additions, and integration value estimates. Tracking synergies separately from valuation allows the team to explicitly document the strategic premium paid for a target.

⚖️ IC tab (Investment Committee)

The IC tab tracks all Investment Committee sessions related to this deal. It records when the deal was presented, who attended, what the outcome was, and any conditions imposed.

IC Session outcomes

  • Approved — IC gave full approval to proceed
  • Rejected — IC declined; deal moves to Dropped/Rejected stage
  • Conditional — Approved subject to specific conditions
  • Deferred — Decision postponed; further information required

Adding an IC session

  1. Click Add SessionOpens the Add IC Session dialog.
  2. Fill in the detailsSession Date, Type (Preliminary Review / Stage Gate / Final Approval / Post-Mortem), Location/Link, Agenda/Notes (rich text), and Participants list.
  3. SaveThe session is recorded with a timestamp.
ℹ️
When you initiate a stage change, Ringside automatically runs a Stage Gate Advisory check and shows a pre-flight banner (pass / warn / block) with a checklist of items to review before confirming the stage move.

👥 Stakeholders tab

The Stakeholders tab captures all the people and organisations involved in the deal — from the target's own leadership to the advisors on both sides of the table.

Counterparty section

A single structured record for the target company: Legal entity name, CIN / Registration number, Founders, Key Management team, Controlling Shareholders, Website, and free-form Notes.

Internal Sponsors

A list of internal Manipal Group champions and sponsors for the deal. Each entry has Name, Role/Title, Email, and Notes. Add, edit, or delete sponsors at any time.

Advisors

Two separate sections — External Advisors (Buy-side) and Counterparty Advisors (Sell-side). Each advisor record includes Advisor Type, Firm Name, Contact Name and Email, Engagement Date, Fee Structure, Conflicts Status (Pending / Cleared / Flagged), and Notes.

⚠️
If any advisor has a Flagged conflicts status, a warning banner appears at the top of the Stakeholders tab as a reminder that a conflicts issue needs resolution before proceeding.

🛡️ Compliance tab

The Compliance tab has two sections for tracking the legal and regulatory obligations of a deal.

NDA Register

A table of all NDA records for the deal. Each record shows: Counterparty, Effective Date, Expiry Date, Scope (One-way / Mutual), Confidentiality Term (months), Status (Active / Expired / Extended), and Document Reference link.

  • NDAs expiring within 30 days are flagged with an amber badge
  • Expired NDAs show in red
  • A global alert banner appears at the top of the tab when any NDA requires attention

Regulatory Clearance Map

📅 Timeline & Audit tabs

The Timeline tab shows the chronological audit trail of all stage changes — when the deal moved from stage to stage, who changed it, and the reason given. The Audit tab provides a broader log of all significant events and field changes on the deal record.

These tabs are read-only and serve as the permanent record of how a deal progressed through the pipeline.

Section 7

Actions Command Center

The Actions Command Center (accessed via Actions in the sidebar) is a pipeline-wide view of every open action item across all deals you have access to. Use it to manage your task queue and that of your team.

🗂️ Action groups

Actions are automatically sorted into collapsible groups based on status and due date:

⚠️ Overdue — past their due date and not completed
🚫 Blocked — marked as Blocked by the owner
📅 Due This Week — due within the next 7 days
🔜 Upcoming — due further in the future
✅ Recently Completed — completed in the last 14 days (collapsed by default)

🔧 Filters

Use the filter bar to narrow the action list:

  • Owner — filter to a specific team member
  • Priority — Critical / High / Medium / Low
  • Must-Win Only — toggle to show only actions on Must-Win deals
  • Overdue Only — toggle to show only overdue actions
  • Search — text search across action descriptions
  • Deal Type — filter by the type of deal the action belongs to
  • My Deals / All Deals — toggle to see your personal queue vs. the full team queue

⚡ Quick actions

Each action card shows the target name, stage, priority tier badge, owner, due date, and priority. Use the Complete button to mark an action done, or Reopen to bring a completed action back to open status. Click the deal name to open that target's detail page.

Section 10

Document Review

The Document Review page (Document Review in the sidebar) aggregates documents across all deals for review governance. It surfaces Must-Win deals with missing required documents and shows documents by their review status.

Each document entry shows: deal name, document title, category, review status, owner, date, and a direct link to the file. Status values are:

  • Requested — document has been requested from the counterparty
  • Received — document received but not yet reviewed
  • Under Review — actively being reviewed by the team
  • Reviewed — review complete
  • Missing — required document not yet received
  • Not Applicable — not required for this deal

Type any question in the chat input. The Copilot responds with pipeline-grounded answers. Suggested prompts at the top give you a quick starting point:

  • "Which deals have overdue actions?"

Section 13

Doctrine

The Doctrine page (Doctrine in the sidebar) is a historical learning record — a systematic review of past deals to extract lessons and improve future decision-making accuracy. It shows concluded deals alongside the initial assessments made at the time and whether those assessments proved correct.

Each entry records: deal outcome (Closed / Dropped / Rejected), the initial scoring and rationale, and an accuracy assessment (Correct / Partially-correct / Wrong) of the team's original judgement. A downloadable Doctrine Report PDF can be generated for offline sharing.

💡
Review the Doctrine page quarterly to build team calibration — understanding where initial assessments were wrong is as valuable as understanding where they were right.
Section 14

Import Wizard

The Import Wizard (Import button in the Pipeline header, or Import in the sidebar) allows you to bulk-upload deal targets from a CSV or Excel file. It walks you through 5 steps:

1. Upload
→
2. Map Columns
→
3. Preview
→
4. Apply
→
5. Done
  1. Upload Drag and drop or browse for a .csv, .xlsx, or .xls file. A raw file preview appears immediately so you can verify the data looks correct before proceeding.
  2. Map Columns Ringside auto-detects which of your spreadsheet columns maps to which deal field. Review and adjust the mapping dropdowns where auto-detection guessed wrong. Set any column to Skip to ignore it.
  3. Preview See a classification of every row: Create (new target), Update (existing target, showing which fields will change), or Skip (row ignored due to missing required data or safety rules).
  4. Apply Click Apply Import. Ringside creates and updates targets row-by-row. Each row succeeds or fails independently — a problem in one row doesn't affect the others.
  5. Done A summary shows how many targets were created, updated, and skipped, plus per-row error details for any rows that failed.

🛡️ Import safety rules

  • Blank values in the import file never overwrite existing non-blank data in the database
  • A target's code identifier (targetCode) is never updated on existing records
  • Rows with invalid tier or stage values are skipped with an explanatory reason
Section 15

Notifications

The notification system automatically monitors your pipeline for conditions that require attention and surfaces them as in-app alerts. Notifications are generated every 15 minutes (checked via a local timer — no extra setup required).

📬 Notification types

TypeTriggerIcon
Stage stagnation An active deal has been in the same stage for more than 45 days with no progression ⏰
Action overdue An open action item has passed its due date ⚠️
NDA expiring An active NDA has an expiry date within the next 30 days 📄
Must-Win no activity A Must-Win deal has had no logged interaction in 14 or more days ⚡

Notifications are deduplicated — you won't get the same alert for the same deal and condition more than once per 24 hours. Clicking a notification navigates directly to the relevant deal and marks the notification as read.

Section 16

Admin Console

The Admin Console (Admin in the sidebar) is only visible to users with the Admin role. It provides user management and deal access control for the entire platform.

🔐
Only Administrators can access the Admin Console. Non-admin users cannot see this section of the sidebar.

👤 User management

The Admin Console shows a list of all platform users with their email address, display name, and current role. From here you can:

ActionHow
Invite a new userClick Invite User. Enter their email, display name, and role. The system sends them an invite email; you can also copy the invite link to send manually.
Change a user's roleUse the role dropdown in the user's row to change their role instantly.
Manage deal accessClick the key icon in a user's row to open the Access dialog — a checklist of all deals. Tick the deals the user should be able to see. Admins bypass this; access grants are for non-admin roles only.
Remove a userClick the delete icon in the user's row and confirm. This removes the user's account and all their sessions.
Generate an OTP for a userUse the OTP button to generate a one-time password for a user who is locked out.

🔑 Roles

RoleCapabilities
AdminFull access to all deals and the Admin Console. Can manage users, roles, and access grants. Sees the entire pipeline regardless of access grants.
Deal LeadCan create and manage deals they have been granted access to. Can add IC sessions.
MemberCan view and interact with deals they have been explicitly granted access to.
IC VoterRead access to granted deals; participates in IC sessions as a voter.

🔒 Deal-level access

By default, non-admin users see no deals until an Admin explicitly grants them access. This ensures that confidential deals are only visible to authorised team members. Admins always see all deals.

When a user creates a new target, they are automatically granted access to it.
