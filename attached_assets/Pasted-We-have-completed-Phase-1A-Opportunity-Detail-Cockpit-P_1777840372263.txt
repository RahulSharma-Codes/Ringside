Phase 3A goal:

Required UX:
1. Add an “AI Copilot” entry point in the app navigation and/or dashboard.
2. Create an AI Copilot page or right-side panel.
3. User can type a question.
4. The app sends the question to a backend AI endpoint.
5. Backend gathers relevant Supabase data and gives the AI a structured context.
6. AI responds in a clean chat-style interface.
7. Show suggested starter questions:
   - Which Must-Win opportunities need attention?
   - Which deals have overdue actions?
   - Summarize the active pipeline.
   - Which opportunities have had no interaction in 30 days?
   - What changed recently?
   - Which deals are stuck by stage?
   - What should I review before the weekly pipeline meeting?

Backend requirements:
1. Add a read-only API route, for example POST /api/ai/ask.
2. The route should require existing app authentication.
3. The route should read from Supabase using existing Drizzle models/routes.
4. Build a compact data context including:
   - targets
   - current stages
   - priority tiers
   - open and overdue actions
   - recent interactions
   - latest stage changes
   - needs-attention flags if available
5. Keep context compact so it does not send excessive data.
6. The model must be instructed:
   - answer only from provided app data
   - say when data is unavailable
   - do not invent targets or facts

8. Prefer OPENAI_API_KEY if present.
