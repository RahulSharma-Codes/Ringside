Phase 2A goal:
Build a safe Excel/CSV Import and Update Wizard so a non-technical user can upload an old M&A tracker and import or update opportunities in Supabase.

Required UX:
1. Add an Import page or Import button accessible from Pipeline.
2. User can upload .csv and, if already supported by dependencies, .xlsx.
3. If .xlsx support requires a new package, inspect package/dependency constraints first and report before installing. CSV support is required in Phase 2A; XLSX support is preferred if safe.
4. Show a preview of uploaded rows.
5. Auto-detect common columns:
   - Target Code
   - Project Name
   - Legal Name
   - Sector
   - Subsector
   - Country
   - Geography / Region
   - Owner
   - Priority / Priority Tier
   - Stage
   - Strategic Rationale
   - Notes
6. Allow user to manually map upload columns to app fields.
7. Validate rows before import.
8. Detect and show:
   - new target rows
   - existing target updates matched by targetCode
   - duplicate target codes inside upload
   - missing target code
   - missing project name
   - invalid priority tier, if validation exists
   - invalid stage, if validation exists
9. Show a review screen before applying changes:
   - rows to create
   - rows to update
   - rows skipped
   - warnings
10. User must click Apply Import before any database writes happen.
