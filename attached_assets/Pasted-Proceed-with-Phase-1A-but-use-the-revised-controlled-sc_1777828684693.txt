Phase 1A goal:
Upgrade the target detail page into a professional mobile-first Opportunity Detail Cockpit.

Required features:

1. Edit Target Overview
- Add an Edit Target modal/drawer on the target detail page.
- Editable fields should include:
  - project name
  - priority tier
  - strategic rationale
  - sector
  - subsector
  - geography/country
  - deal owner
  - deal champion
  - executive sponsor
  - score fields if already supported by current schema/API
- Do not add migrations. If a requested field does not exist in the current schema, skip it and report it.

2. Edit Interactions
- Each interaction card should have an edit icon.
- Clicking edit opens a pre-filled modal.
- Editable fields:
  - type
  - summary/notes
  - participants internal, if existing schema supports it
  - participants external, if existing schema supports it
  - sentiment, if existing schema supports it
  - valuation signal, if existing schema supports it
- Saving should PATCH/PUT the interaction and refresh the interaction list.
- Do NOT add delete interaction in this phase unless it already exists safely. Prefer no delete for auditability.

3. Add Interaction Form Improvements
- Existing Add Interaction form should expose:
  - type
  - summary/notes
  - participants internal, if supported
  - participants external, if supported
  - sentiment: Positive / Neutral / Negative, if supported
  - valuation signal, if supported
- If fields are not in the current schema, do not create migrations. Report skipped fields.

4. Edit Actions
- Each action card/row should have an edit icon.
- Clicking edit opens a pre-filled modal.
- Editable fields:
  - title/description, matching current schema
  - owner
  - due date
  - priority
  - status
- Saving should update the action and refresh the action list.

5. Complete/Reopen Actions
- Each action should show:
  - Complete button when status is not Completed
  - Reopen button when status is Completed
- Complete should set status to Completed.
- Reopen should set status to Open.
- Keep this inline without leaving the detail page.

6. Clickable Hyperlinks
- Create a small reusable LinkifiedText React component.
- It should detect URLs in interaction notes/summaries and action descriptions.
- Render URLs as clickable links with target="_blank" and rel="noopener noreferrer".
- Apply it wherever notes/descriptions are displayed.

7. Change Stage Modal with Audit Log
- Add a Change Stage button on the target detail page.
- Modal must show current stage.
- Modal must allow selecting a new stage from the existing stage list or accepted stage values.
- Modal must require a reason/note.
- On submit:
  - update the current stage using the existing database model
  - insert a timestamped row into stage_change_log
  - refresh target detail and stage history
- Do not bypass stage_change_log.

8. Mobile-first UI
- Make the target detail page readable on a 375px mobile screen.
- Use card-based layout.
- Avoid cramped tables on mobile.
- Add clear section headers.
- Use stage chips, priority chips, and action status badges.
- Add a sticky bottom action bar on mobile with:
  - Log Interaction
  - Add Action
  - Change Stage

- Only touch OpenAPI/codegen if the app is actually using generated hooks for these flows.
- If OpenAPI/codegen is used, run the relevant codegen and confirm generated files compile.
