5. Backend routes
Add routes only if needed, likely:
- POST /api/documents/:id/upload
- GET /api/documents/:id/download-url
- PUT /api/documents/:id/replace-file
- GET /api/documents/:id/file-info if needed

Behavior:
- Upload route receives multipart/form-data.
- Backend validates document exists.
- Backend stores file under path pattern:
  targets/{targetId}/documents/{documentId}/{safeFileName}
- Backend updates document metadata after successful upload.
- Download route returns a short-lived signed URL if using private Supabase Storage.
- Replace route uploads new file and updates metadata.
- Do not delete old files in Phase 5B unless replacement cleanup is safe and explicitly implemented.
- Do not add hard delete.
